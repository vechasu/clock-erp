# Аудит регрессии страницы «Товары» после PR #9

Сравнение выполнено между `eb56e33b3059af6969a45ea80d2d1c48b5c16283`
и `530cb789ae4e164f439d1da1cb20bc9629e20b4c`.

## Точная причина

`app/templates/warehouse.html` между этими commit не изменился: файлы
побайтно совпадают. Регрессия появилась потому, что пункт навигации «Товары»
был переключён с `/warehouse` на новый `/products`. Новый маршрут отрисовывал
отдельный шаблон `excel_products.html` длиной 75 строк с обычной GET-формой и
серверной пагинацией по 50 строк. Старые CSS/HTML/JavaScript-блоки остались в
`warehouse.html`, но перестали быть частью основной страницы «Товары».

## Изменения backend-параметров

До PR #9 `/warehouse` принимал `q`, `brand`, `category`, `cell`, `hide_zero`,
`sort_by`, `sort_dir` и `refresh`. После PR #9 `/products` принимал только `q`,
`brand`, `category`, `match_status`, `page`, `per_page`; сортировка была всегда
по `excel_row`.

Исправленный `/products` принимает `q`, `brand`, `category`, `cell`,
`match_status`, `hide_zero`, `sort_by`, `sort_dir`, `page`, `per_page` и
служебный read-only `_partial`. Все sort/status значения проходят whitelist.
Категория совпадает с выбранным узлом или его дочерними путями.

## Какие блоки перестали использоваться

Изменение навигации исключило из основной страницы следующие неизменённые
блоки `warehouse.html`:

- CSS: `.search-card`, `.search-input-wrap`, `.search-clear-button`,
  `.zero-stock-toggle`, `.brand-combobox`, `.warehouse-bulk-form`,
  `.warehouse-map-grid`, `.stock-zero`, `.column-resize-handle` и mobile
  `@media` правила;
- HTML: `warehouseSearchInput`, `warehouseSearchClear`, `warehouseBulkForm`,
  `warehouseAddCard`, `filterDrawer`, `warehouseMapDrawer`, компактная таблица
  с колонками brand/category/stock/created_at/cell/actions;
- JavaScript: `filterWarehouseRows`, `clearWarehouseSearch`, debounce 350 мс,
  `syncWarehouseBulkSelection`, `toggleZeroStockRows`, brand combobox,
  `sortWarehouseTable`, `warehouse-resizable-columns-script` и восстановление
  `warehouseScrollPosition`.

Новый `excel_products.html` PR #9 содержал только `handleProductImageError`;
поиск и фильтры были обычной GET-формой с кнопкой «Применить».

## Граница операций записи

Основной `/products` показывает Excel-authoritative каталог и выполняет только
чтение при поиске, фильтрации и сортировке. Старые операции создания и массового
изменения относятся к операционному складу МойСклад и поэтому не применяются к
Excel ID автоматически: интерфейс даёт явные ссылки на сохранённые формы
`/warehouse`. Это предотвращает смешивание внутренних Excel `product_id` с UUID
МойСклад. В ходе разработки и проверки формы записи не отправлялись.
