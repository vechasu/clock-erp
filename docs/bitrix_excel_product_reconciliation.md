## Консервативный алгоритм

Инструмент `scripts/reconcile_bitrix_excel_catalog.py` открывает SQLite только в
режиме `mode=ro&immutable=1`, читает Excel и создаёт JSON/CSV dry-run. Он не
импортирует Flask-приложение, внешние клиенты или функции прихода.

Индексы строятся по ID Bitrix, `XML_ID`, артикулу, точному нормализованному
названию, бренду + названию и бренду + названию без ведущего бренда. Для
исследуемого Excel первые три индекса не дают совпадений из-за отсутствия ID в
Excel и SKU в Bitrix.

Автоматические статусы ограничены двумя случаями:

1. `exact` — единственная карточка с тем же нормализованным названием внутри
   того же нормализованного бренда;
2. `high_confidence` — то же условие после безопасного удаления бренда из
   начала названия Excel.

Нормализация меняет регистр, Unicode-дефисы, повторные пробелы, слеши,
амперсанды и знаки пунктуации. Цвета, размеры и значимые слова не удаляются.
Поиск похожих названий и токенов модели используется только для списка ручных
кандидатов и никогда не даёт автоматический статус.

Автоматическое решение запрещается при различии бренда, общем однословном
названии (`Black`, `Gold`, `Moon`, `Eclipse` и аналоги), нескольких карточках и
различии признаков образца/комплекта. Различия цвета и размера остаются в
названии, поэтому похожие варианты попадают только на ручную проверку.

Идентичность внутренней складской карточки определяется номером строки Excel,
а не нормализованным названием или артикулом. Поэтому одинаковые названия не
объединяются, остатки не суммируются, а исходные поля и различия каждой строки
сохраняются отдельно. Если несколько строк безопасно сопоставлены одной
карточке Bitrix, связь явно получает тип `many_to_one`: контент можно показать
у каждой складской карточки, но остатки остаются независимыми. Совпадение лишь
ручных кандидатов помечается `many_to_one_candidate` и контент не подключает.

## Реализованная внутренняя модель

Excel является единственным источником состава, фактического остатка и ячейки.
Bitrix используется только как read-only источник снимка контента при
подтверждённой связи. МойСклад не участвует в применении batch; у карточек и
партии сохраняется `moysklad_sync_status = not_linked`.

В `catalog.db` добавлены пять внутренних сущностей:

1. `catalog_excel_batches` — уникальный SHA-256, стабильный `batch_id`, источник,
   контрольные суммы строк и остатков, предшествующая партия и статус;
2. `catalog_excel_products` — одна активная карточка на устойчивый ключ номера
   строки Excel, исходные поля, раздельный остаток и отдельный снимок обогащения
   Bitrix, включая явную cardinality связи;
3. `catalog_excel_batch_rows` — старое и применённое состояния каждой строки;
4. `catalog_excel_stock_operations` — корректировки «старое → новое» и обратные
   операции rollback только для ненулевой разницы;
5. `catalog_excel_match_audit` — ручная привязка, отметка «В Bitrix нет», отмена
   связи и undo.

Apply выполняется одной SQLite-транзакцией. Для каждой строки рассчитывается
`difference = excel_stock - old_stock`, после чего внутренний остаток
устанавливается ровно в `excel_stock`. Поэтому 0 — полноценное целевое значение,
но движение с разницей 0 не создаётся. Для первой загрузки планируется 836
ненулевых корректировок; dry-run не применяет их, поэтому фактических движений
0. Повтор того же SHA-256 возвращает существующую партию без новых
операций.
Rollback разрешён только для активной партии: он восстанавливает сохранённые
поля и остатки предыдущей партии, удаляет только созданные партией карточки без
ручной истории или использования другими партиями и пишет обратные операции
только для ненулевых возвратных корректировок.

Команда `scripts/apply_excel_product_catalog.py` теперь создаёт только временный
draft прихода. Прямой `--apply` отключён: проведение выполняется только явным
POST-подтверждением в интерфейсе «Оформить приход». Одинаковые названия или общая карточка Bitrix не блокируют
batch; только `invalid` останавливает всю транзакцию до записи. Для отката
используется `--rollback-batch <batch_id>`.

Интерфейс `/products` по умолчанию показывает только активный Excel batch.
Bitrix-only карточки доступны отдельно через `/catalog`. Фильтр «Требует
сопоставления» показывает `ambiguous`, а карточка товара позволяет выбрать
кандидата, отметить отсутствие в Bitrix, отменить связь или вернуть последнее
ручное решение. Подтверждение меняет только локальное обогащение; Excel-остаток
и ячейка не изменяются.
