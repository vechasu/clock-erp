# Реальная структура каталога Bitrix tictactoy.ru

Исследование выполнено 20.07.2026 штатными read-only API Bitrix через
`scripts/bitrix_catalog_server_diagnostic.php`. Запускался PHP 8.3.17 от
пользователя сайта `admin`; версия «1С-Битрикс: Управление сайтом» —
23.900.1000. База и файлы сайта не изменялись, секреты и персональные данные не
читались и не выводились.

## Сервер и исходный код

* Корень сайта: `/var/www/admin/data/www/tictactoy.ru`.
* Проект сайта не является Git-репозиторием. В корне есть `.gitignore` и
  `.git-ftp.log`, но каталога `.git` нет; commit/push проекта Bitrix невозможен.
* Рабочий CLI: `/opt/php83/bin/php`. Системный PHP 7.4 несовместим с текущим
  ядром Bitrix.
* Основной каталог жёстко указан как `IBLOCK_ID=5` в `catalog/index.php` и
  `catalog/detail.php`.

## Торговые предложения

SKU-инфоблока нет. Для каталога 5 `PRODUCT_IBLOCK_ID=0`, `SKU_PROPERTY_ID=0`,
а `CCatalogSKU::GetInfoByProductIBlock(5)` не возвращает связь. Количество
торговых предложений — **0**. Вариации фактически заведены отдельными элементами
основного инфоблока; свойства `CHOSE_COLOR` и `CHOSE_SIZE` не образуют штатную
модель торговых предложений.

Дочерние разделы преимущественно являются брендами, но среди них есть служебные
проморазделы (`SALE`, `Бестселлеры`, `WOW-Цена`, архивные акции). Основное
значение бренда хранится в списочном свойстве 86 `BRAND_MODEL` («Марка часов»),
которое заполнено у 3872 элементов. Для экспорта `brand` берётся из
`BRAND_MODEL`; раздел не должен безусловно считаться брендом.

Товар имеет основной `IBLOCK_SECTION_ID`, а дополнительные привязки должны
выгружаться через `CIBlockElement::GetElementGroups`. Endpoint возвращает все
связи, полный путь и отдельно `primary_category_id`.

## Изображения и галерея

* `PREVIEW_PICTURE` заполнена у всех 4684 элементов.
* `DETAIL_PICTURE` заполнена у 238 элементов.
* Свойство 18 `GALLERY` — множественное файловое свойство: 4671 товар,
  21 944 файловых значения.
* Свойство 35 `IMAGE` — одиночная картинка главной/бренда: 441 товар.
* Свойство 99 `SHOW_IN_MENU_PICTURE` — одиночная картинка меню: 6 товаров.
* Свойство 161 `PREVIEW_PICTURE2` — вторая картинка анонса: 1580 товаров.

Все пять источников нужно объединять, сохраняя тип и порядок. Дубли удаляются по
ID файла, затем по нормализованному абсолютному URL. Оригинальные метаданные
получаются через `CFile::GetFileArray`; base64 не используется.

## Артикул, штрихкод, XML_ID и складские поля

Отдельного свойства артикула в инфоблоке 5 нет. В экспортном контракте
`article` поэтому остаётся пустым; нельзя автоматически подменять его `CODE` или
`XML_ID`. `CODE` является символьным кодом URL, `XML_ID` заполнен у всех 4684
товаров и выгружается отдельно.

Штатная таблица штрихкодов каталога не содержит записей для инфоблока 5:
`barcode_rows=0`, поэтому `barcode` будет пустым. Не следует придумывать его из
артикула или XML_ID.

`CCatalogProduct` содержит строки для всех товаров. Ненулевой `QUANTITY` есть у
4359 товаров, `QUANTITY_RESERVED` — у 2112, мера — у 4358. Вес и габариты не
заполнены. Доступны стандартные поля `QUANTITY`, `QUANTITY_RESERVED`,
`AVAILABLE`, `WEIGHT`, `WIDTH`, `LENGTH`, `HEIGHT`, `MEASURE`; они служат только
для диагностики/отдельной синхронизации остатков и не импортируются в контентный
каталог Vechasu. Мера по умолчанию — ID 5, код 796, «Штука» (`шт`).

## Описания, даты и SEO

Доступны стандартные поля элемента `PREVIEW_TEXT`, `PREVIEW_TEXT_TYPE`,
`DETAIL_TEXT`, `DETAIL_TEXT_TYPE`, `DATE_CREATE`, `TIMESTAMP_X`, `ACTIVE`,
`SORT`, `CODE`, `XML_ID` и `DETAIL_PAGE_URL`. SEO нужно получать через
`\Bitrix\Iblock\InheritedProperty\ElementValues`, включая вычисленные TITLE,
KEYWORDS и DESCRIPTION, а свойство `SEO_TEXT` экспортировать также как обычное
свойство.

`TIMESTAMP_X` надёжно отражает изменение основной карточки и свойств элемента,
но изменение `CPrice` и складской строки может не менять его. Поскольку штатных
SKU нет, offer timestamp не требуется. Для `updated_from` endpoint использует
консервативную стратегию: фильтрует товары по `TIMESTAMP_X`, но документация API
требует периодический полный проход; изменение только цены может быть замечено
лишь полным проходом.

## Итог для экспортного endpoint

Источник — только инфоблок 5. Endpoint должен включать активные и, по флагу,
неактивные элементы; все 56 свойств; все 97 разделов и дополнительные связи;
пять источников изображений; цену типа 1; стандартные catalog-поля без импорта
остатков; вычисленные SEO-поля. `offers` всегда пустой массив, `article` и
`barcode` пусты, пока в Bitrix не появится явный источник этих данных.
