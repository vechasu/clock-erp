## Очистка

Импортёр исправлен: новые строки создаются только при непустом `value` или
`display_value`. Preview показал 121315 удаляемых строк и не изменил БД.

Перед apply создана копия:

`/var/backups/clock-erp-catalog/20260720-160141-property-cleanup/catalog.db`

Удалено ровно 121315 строк с `null/null`. После очистки осталось 140989 строк,
повторный preview показывает 0 удаляемых строк. `PRAGMA quick_check` вернул
`ok`, `PRAGMA foreign_key_check` — 0 ошибок. Товары и определения 56 свойств не
удалялись.

## Контрольный запрос МойСклад

Production-проверка использовала endpoint без токена в тексте отчёта:

`https://api.moysklad.ru/api/remap/1.2/entity/product`

Параметры: `limit=1`, `offset=0`; фильтров по названию, архивности, складу или
остатку не было. Измерялось поле `meta.size` — количество сущностей `product`,
доступных текущему production-токену. Это не количество единиц на остатках.

Production-токен вернул 0 для `product`, `variant`, `bundle`, `service` и
`assortment`, а также 0 дополнительных атрибутов товара. Поэтому прежнее
`meta.size = 0` технически корректно для production-учётной записи, но означает,
что она не содержит доступного ассортимента и не подходит для полезного
сопоставления.

Отдельный read-only запрос с локальной development-конфигурацией вернул 160
товаров. Это другая доступная учётная запись/credential context. На её тарифе
получение metadata дополнительных полей запрещено; в самих строках товаров
отдельное поле `XML_ID` не обнаружено. Стандартный `externalCode` доступен и
обрабатывается отдельно. Production-секреты автоматически не заменялись.
