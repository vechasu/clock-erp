# Реальный dry-run каталога Bitrix

Дата: 20.07.2026. Запуск выполнен на production Vechasu:

```bash
LANG=en_US.utf8 LC_ALL=en_US.utf8 \
venv/bin/python scripts/bitrix_catalog_dry_run.py --limit 20
```

Токен и конфигурация в отчёт не включены. Имена и технические ID отдельных
карточек не сохранялись; ниже только агрегаты.

Endpoint вернул реальную структуру инфоблока 5: все товары выборки имеют
XML_ID, бренд, категории, изображения, свойства и цену BASE/RUB. Артикулы
отсутствуют в самом Bitrix, что совпадает с серверным исследованием. SKU нет,
вариации являются самостоятельными карточками.

Read-only запрос МойСклад вернул `meta.size=0` и ноль строк для текущего токена,
поэтому все 20 карточек классифицированы как новые. Проверены варианты запроса с
`limit=1`, `limit=1000` и `expand=attributes`; во всех случаях API вернул пустой
список без HTTP-ошибки. Это не подменялось фиктивными совпадениями.

## Проверка отсутствия записей

До и после запуска сравнивались SHA-256 всех файлов верхнего уровня `instance/`:
изменений нет. `instance/catalog.db` отсутствовала до запуска и не появилась
после. Скрипт импортирует из клиента МойСклад только метод `GET`; методы POST,
PUT, DELETE, складские операции и создание товаров не вызываются. Git-репозиторий
production остался без изменений файлов.

Первый технический запуск остановился до сетевых запросов из-за ASCII-локали
Python 3.6 при чтении существующего `.env`. Повторный запуск выполнен с доступной
серверной локалью `en_US.utf8`; `.env` для этого не изменялся.
