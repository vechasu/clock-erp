# Проверка production endpoint каталога Bitrix

Дата: 20.07.2026. URL без токена:
`https://www.tictactoy.ru/api/catalog-export.php`.

Endpoint установлен как новый файл с владельцем `admin:admin` и режимом 0640.
Секрет хранится вне webroot в локальном PHP-конфиге с режимом 0600 и не
выводился в команды, ответы или логи. Проект сайта не находится под Git;
проверенный исходник endpoint хранится в репозитории Vechasu в
`bitrix/catalog-export.php`. Перед установкой создан deployment manifest в
`/var/backups/tictactoy-catalog-export/20260720-145531`; резервная копия самого
endpoint не требовалась, потому что файл был новым.

## Реальные HTTP-проверки

| Сценарий | Результат |
|---|---|
| `mode=meta` | HTTP 200; инфоблок 5; 4684 всего, 1550 активных, 3134 неактивных; 56 свойств; 0 SKU |
| `page=1&limit=2` | HTTP 200; ровно 2 товара; total 1550 |
| `page=1&limit=100` | HTTP 200; 100 товаров; 0,330 с; 2 209 324 байта |
| последняя активная страница | HTTP 200; page 16; 50 товаров; `has_more=false` |
| неверный токен | HTTP 401; `{"error":"unauthorized"}` |
| отсутствующий токен | HTTP 401; `{"error":"unauthorized"}` |
| `page=0` | HTTP 400; `invalid_page` |
| `limit=201` | HTTP 400; `invalid_limit` |
| `updated_from=2099-01-01...` | HTTP 200; пустая корректная выборка |
| `include_inactive=1&limit=200` | HTTP 200; total 4684; в выборке есть неактивные товары |
| несколько изображений | подтверждено; максимум в проверенной странице — 27 |
| много свойств | подтверждено; максимум непустых свойств — 39, определения всех 56 присутствуют |
| цены | у всех 100 проверенных товаров есть цена |
| торговые предложения | `offers=[]`, meta `offers_total=0`, соответствует структуре каталога |

Ответ `limit=100` укладывается в разумное время и содержит товары вместе со
свойствами, разделами, файлами, ценами и SEO без дополнительных HTTP-запросов.

## Проверка неизменности Bitrix

Непосредственно до и после HTTP-тестов повторно выполнена агрегированная
read-only диагностика. Сравнивались количества элементов, активность, разделы,
заполненность стандартных полей и всех свойств, catalog storage, типы цен и
валюты. Контрольный SHA-256 агрегированного состояния совпал:

`171342001f2ff332e9894340360459dbae1599a609af6918b3fee112de99851d`

Endpoint не изменил товары, свойства, цены, остатки или другие данные Bitrix.

## Известное ограничение `updated_from`

Фильтр использует `TIMESTAMP_X` элемента. Изменение только строки цены или
catalog quantity может не изменить `TIMESTAMP_X`; endpoint сообщает это в
`updated_from_scope`. Для контента требуется периодический полный контрольный
проход. Остатки импортёром Vechasu обрабатываться не будут.
